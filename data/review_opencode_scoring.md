# 다중 AI 스코어링/블라인드 검증 방법론 검토 (OpenCode 리뷰)

> 범위: ai_scoring_v2.py, multi_round_scoring.py, blind_coding.py, crisis_scoring.py
> 버전: 2026-03-28

본 문서는 본 저장소의 다중 AI 스코어링 설계와 블라인드 검증 프로토콜을 코드 수준에서 점검한 결과이다. 각 방법론별로 (a) 강점, (b) 약점/위협, (c) 개선 권고를 제시한다.

---

## 1) 다중 AI 스코어링 설계 검토 (ai_scoring_v2.py, multi_round_scoring.py)

요약: 5개 LLM을 독립 코더로 활용하여 ACW(−2~+2)·제도적 메커니즘(0~2)·Lowi 분류를 채점하고, 패널1(중앙정부)·패널2(지방정부)를 병행한다. v2에서 JSON 강제·검증을 강화했고, multi_round_scoring.py로 5라운드(1회 + 추가 4회) 반복 측정과 표준편차 산출을 수행한다.

a) 강점
- 모델 다양성: GPT-5.2, Claude Sonnet 4.6, Grok-4-1, Gemini 3 Flash, HyperCLOVA HCX-007의 이질적 조합으로 단일 모델 편향을 구조적으로 완화.
- 견고한 파이프라인: JSON 강제/복구(extract_json, _repair_json), 검증(validation)과 재시도(retry_gaps), 건 단위 저장(중단 복구) 등 운영 안정성 확보.
- 반복 측정 설계: 5라운드 평균 + 기준별 표준편차(std) 기록으로 측정 안정성 가시화. CLOVA의 thinking 비활성화(반복 시)로 분산 축소.
- 비교 가능성 관리: 패널2(지방)에서 중앙 전용 기준을 명시적으로 제외(sub-wheel)하고, 중앙-지방 직접 비교 불가를 보고서에 명시.
- 신뢰도 산출: 모델 간 합의(consensus 평균)와 Fleiss’ kappa를 계산하여 코더 간 일치도를 제시.

b) 약점/위협
- 신뢰도 척도의 수준 불일치: ACW(연속적·서열적 척도), 제도 점수(서열/등간적)에 대해 Fleiss’ kappa(명목형 전제)를 사용. 정보 손실 및 해석 왜곡 위험.
- 0점(중립) 억제 규칙의 잠재 편향: multi_round에서 “판단유보 금지” 지침과 0점 과다 시 검증 실패 규칙이 결합되어, 실제 중립 분포를 저평가하거나 양극화된 점수 분포를 유도할 위험.
- 샘플링 설정 비일관: ai_scoring_v2는 T=0.3, blind_coding은 T=0.2. top_p 등 추가 샘플링 파라미터 미통제. CLOVA만 thinking 제어(패널1 기본 실행에서는 medium), 반복 측정에서는 비활성화 → 라운드 간·모델 간 분산 원인 혼재.
- 앵커 기반 교정 부재(스코어링 측정계): blind_coding에는 앵커링 비네트가 있으나, 본 스코어링 라인에는 교정 문항이 없어 모델별 절편/기울기 차이(관대/엄격 경향) 교정이 미흡.
- 이질적 모델(HCX-007) 처리: 민감도 분석(포함/제외), 가중 평균(신뢰도 기반) 등 견고성을 코드 수준에서 기본 제공하지 않음. 합의는 단순 평균에 치우침.
- 재시도 정책의 측정 간섭 가능성: JSON 파싱 실패나 코드 누락 시 재시도하는 구조가 ‘유효성 통과’에 유리한 샘플을 선택할 가능성. 1차 응답 보존·표지 및 재시도 사유 구분 필요.
- 패널1 LJM 컨텍스트 특이적 주입: build_acw_prompt에서 이재명 정부만 지방 경력 컨텍스트(local_context)를 조건부 주입 → 정보 비대칭에 따른 체계적 편의 가능성.

c) 개선 권고
- 신뢰도/합의 통계 개선:
  - ACW·제도 점수에는 Krippendorff’s α (interval/ordinal) 또는 ICC(2,k)를 병행 제시. 명목형 kappa는 보조로 유지.
  - 부트스트랩(라운드×모델 재표집)으로 합의 평균과 신뢰구간(CI)을 산출.
  - 합의치는 평균+중앙값(또는 20% 절사평균) 병기, 이상치(MoE/특정 모델) 강건성 확보.
- 모델 교정과 가중치:
  - blind_coding의 앵커링 정확도를 모델 신뢰 가중치로 연결(예: 정확 3/3=1.0, 2/3=0.7…)하고, 합의 평균과 신뢰도 계산에 반영.
  - 모델·차원별 z-정규화/랜덤효과 보정으로 관대/엄격 편향을 교정.
- 샘플링 및 추론 모드 정렬:
  - 전 모델 동일한 temperature(권장 0.1~0.2)·top_p(예: 0.8)·thinking 모드(전부 off 또는 전부 on)를 사전정의. 반복·본선 라인에서 동일 적용.
  - ai_scoring_v2의 CLOVA thinking=medium을 반복 라인과 일치(비활성)시키거나, 양 라인 모두 동일 모드로 고정.
- 재시도 정책 보완:
  - ‘내용 실패(비어있음/코드 누락)’와 ‘형식 실패(JSON 파싱)’를 구분. 형식 실패만 재시도, 내용 실패는 1차 응답을 기록하고 별도 플래그 처리.
  - 1차 응답과 최종 채택 응답을 모두 저장해 선택 편향을 감시.
- LJM 컨텍스트 주입 공정화:
  - 패널1 전체 정부에 대해 동등한 컨텍스트 주입 원칙 수립(예: 각 정부 팩트시트 길이 상한·출처 기준) 또는 컨텍스트 비주입 일원화.
- 민감도 분석 기본 포함:
  - (i) 5모델 vs 4모델(HCX-007 제외), (ii) 모델 신뢰 가중 vs 무가중, (iii) 평균 vs 중앙값 결과를 보고서 표준 섹션으로 상시 제공.

## 2) 블라인드 검증 프로토콜 검토 (blind_coding.py, crisis_scoring.py)

요약: blind_coding.py는 Level 2 블라인딩(대통령명/정당/연도/특정 법률·사건명 치환)을 수행한 뒤, 5모델×3라운드 분류(T/I/D/E)와 식별 테스트를 실시하며, Krippendorff’s α(명목형)를 산출한다. crisis_scoring.py는 타임라인을 이름만 그리스 문자로 치환하는 비교적 약한 블라인딩을 적용해 3차원(0~4) 점수를 산출한다.

a) 강점
- 블라인딩의 공학적 구현: Level 2에서 이름·정당·연도·법률·사건명을 일반화/치환하고, 다중 정규식을 활용하여 잔존 단서를 상당 부분 제거.
- 식별 테스트 내장: 레이블맵 무작위화와 ‘1/6 무작위 기준’ 대비 정확도 판정으로 블라인딩 성공 여부를 계량적으로 확인.
- 앵커링 비네트: 교정 문항(V1~V3)으로 모델별 분류 성향을 사전 점검하고, 가중치(정답수→가중) 도출의 기반을 제공.
- 라운드 합의: 3라운드 다수결로 모델 내부 변동성을 완화하고, 연구자 코드 대비 per-model 일치율을 산출.
- Krippendorff’s α 적용: T/I/D/E(명목) 분류에 적합한 신뢰도 지표를 사용하고, 연구자 포함/제외 α를 병기.

b) 약점/위협
- 위기 채점의 블라인딩 수준 불충분: crisis_scoring.py는 대통령명만 그리스 문자로 치환(Level 1에 가까움). 사건명/연도/정책 고유명사로 정부 식별 가능성이 잔존.
- 가중치의 제한적 활용: 비네트 정확도 가중은 표에만 반영되고, α 계산/최종 합의에는 비반영. 신뢰도 기반 가중 합의 미적용.
- 라운드 집계 방식 단순: 3라운드 ‘다수결’만 사용. 동률·불일치 해소 규칙, 라운드 간 일관성 지표, 부트스트랩 CI 부재.
- α의 불확실성 미제시: Krippendorff’s α에 대한 신뢰구간/유의성 검정 부재.
- HCX-007 편차 처리: 모델 간 구조적 편차(HCX-007 divergence) 가능성에 대한 포함/제외 민감도 분석이 자동화되어 있지 않음.

c) 개선 권고
- 위기 채점 블라인딩 강화 및 검증:
  - crisis_scoring.py에도 Level 2 규칙(연도 일반화, 고유사건/법률명 치환, 정당/이념 제거)을 이식.
  - 동일한 식별 테스트(무작위 레이블맵)와 판정 기준(1/6, 1/3, 0.5)을 추가하여 블라인딩 성공을 계량 확인.
- 신뢰 가중 합의 채택:
  - 앵커링 비네트 정확도에서 산출한 모델 가중치를 분류/채점 합의와 α 산출(모델별 가중 α 또는 재표본화 가중)에 반영.
  - 모델별 분류 ‘신뢰도 프로파일’을 저장·재사용(세션 간 일관 가중).
- 라운드·불확실성 처리 고도화:
  - 라운드별 분류/점수의 표준편차, 일치율, 교차 엔트로피 등 변이 지표를 계산·보고.
  - 부트스트랩(항목×라운드)으로 α와 식별률의 신뢰구간 제시.
- HCX-007 민감도 표준화:
  - blind_coding/crisis_scoring 모두에서 (i) 5모델, (ii) 4모델(HCX 제외) 결과를 병렬 산출·보고.
  - 이상치 영향 진단(leave-one-model-out)과 강건 합의(중앙값/절사평균)를 동시 제시.
- 분류 신호의 잔존 단서 제거 규칙 확대:
  - 어휘 n-그램 블랙리스트(정권/브랜드 명칭), 연도 패턴 정규화, 사건 고유명 매핑의 확장 사전 도입.

## 3) 온도(temperature)·반복 측정·α 산출 방식의 적절성 평가

- Temperature(0.2~0.3): 분류·채점과 같은 규범적 태스크에서는 0.1~0.2 범위가 일반적으로 일관성에 유리. 본 저장소는 blind_coding(0.2)과 스코어링(0.3)이 상이하여, 전 라인 정렬(예: 0.2 고정)이 바람직하다. top_p(예: 0.8) 병기 권고.
- 5라운드 반복: 평균과 표준편차 제시는 신뢰성 향상에 유익. 추가로 중앙값/절사평균을 병기하고, 라운드 수가 부족한 항목에 대한 가중(가용 라운드 수 반영)을 명시하면 견고성 제고.
- Krippendorff’s α: blind_coding의 명목형 α 적용은 적절. 스코어링(등간/서열)에 대해서는 α(interval/ordinal) 또는 ICC가 더 적합하며, CI를 병기하여 해석 신뢰도를 높일 필요.
