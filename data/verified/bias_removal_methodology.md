## 2. 제1층: Action Inventory (사실 기록)

### 원칙
점수를 부여하지 않고, 관찰 가능한 행위만 기록한다.

### 기록 항목

| 항목 | 측정 | 출처 |
|------|------|------|
| 법률 제·개정 | 건수 + 목록 | law.go.kr (verified/laws/) |
| 예산 변동 | 전년 대비 Δ% | e-나라지표, KISTEP |
| 조직 신설/폐지 | 건수 + 목록 | 정부조직법 개정 이력 |
| 위기 대응 시간 | 일수 | verified/policy_lag_verified.md |
| 국제지표 원점수 | 임기 시작/종료 시점 | verified/wgi_korea.md 등 |
| 국정과제 | 건수 + Lowi 분류 | verified/lowi_classification.md |

### 이 층의 역할
어떤 주관적 판단도 포함하지 않음. 누가 보아도 동일한 결과.

## 3. 제2층: T/I/D/E Attribution (귀인 분류)

### 분류 체계

| 코드 | 의미 | 조작적 기준 | 비교 대상? |
|------|------|------------|:---------:|
| **T** (Trend) | 시간이 지나면 자동 발생 | 전임 정부에서 이미 입법/계획됨, 또는 글로벌 추세와 동조 | 제외 |
| **I** (Intentional) | 대통령의 명시적 결정 | 새 입법, 조직 개편, 예산 재배분의 명확한 증거 | **포함** |
| **D** (Drift) | 제도 존재하나 작동 실패 | 동일 제도가 전임에서 작동했으나 해당 임기에서 실패 | **포함** |
| **E** (External) | 대통령 통제 밖 외부 충격 | 글로벌 위기, 기술 패러다임 전환 | 제외 |

### 비교 공식

```
기존: 총점 = Stock(누적) + Flow(기여) → 후임 유리
제안: 비교 대상 = I(의도적 행위) + D(표류)만 추출
      T(추세)와 E(외부)는 맥락으로 보고하되 비교에서 제외
```

### 코딩 절차

1. 제1층의 모든 Action에 대해 연구자가 T/I/D/E 코딩
2. 코딩 기준을 논문에 명시 (재현 가능성 확보)
3. 경계 사례(borderline case)는 명시적으로 표기
4. 제3층에서 AI 검증

### 적용 예시

| 행위 | 정부 | 코드 | 근거 |
|------|------|:----:|------|
| 정부업무평가기본법 존속 | 전 정부 | T | 2006년 제정, 이후 자동 존속 |
| 규제샌드박스법 제정 | 문재인 | **I** | 신규 입법 (2019) |
| 감염병 체계 전면 개편 (MERS→코로나) | 문재인 | **I** | 의도적 학습 |
| 국민청원 폐지 | 윤석열 | **I** (부정) | 의도적 퇴행 |
| 정부3.0 형식적 운영 | 박근혜 | **D** | 제도 존재하나 실질 약화 |
| COVID-19 발생 자체 | — | E | 외부 충격 |
| EGDI 상승 (기술 발전) | 전 정부 | T | 글로벌 기술 추세 |
| AI기본법 시행 + 하위법령 정비 | 이재명 | **I** | 신규 시행 (법률은 21대 국회 통과) |

## 4. 제3층: Blind AI Validation (블라인드 검증)

### 역할
AI를 **평가자(scorer)**가 아닌 **검증자(validator)**로 사용.
연구자의 T/I/D/E 분류가 정치적으로 편향되지 않았음을 검증.

### 블라인딩 수준
- Level 1 (이름 제거): 식별 위험 높아 **부적합**
- **Level 2 (맥락 추상화)**: 날짜, 정당, 특정 사건명 제거 → **권장**
- Level 3 (기준별 단독): D(표류) 판별에만 적합

**권장: Level 2 + Level 3 하이브리드**

### AI 프롬프트 설계 (3종)

**Task 1: T/I/D/E 귀인 검증**
```
다음은 익명화된 정부의 정책 행위입니다. 각 행위에 대해:
(a) T(시간 추세), I(의도적 결정), D(제도 표류), E(외부 충격) 중 분류
(b) 판단 근거를 2문장 이내로 제시

[대통령명, 연도, 정당 정보 제거된 행위 목록]

JSON 형식으로 응답: {"action_id": "A1", "classification": "I", "rationale": "..."}
```

**Task 2: 행위 평가 (적응적 거버넌스 영향)**
```
다음 정책 행위가 적응적 거버넌스 역량에 미치는 영향을 평가하시오.
- 강화(+1) / 중립(0) / 약화(-1) 중 선택
- 판단 근거를 2문장 이내로 제시

주의: 총점을 산출하지 마시오. 개별 행위 단위로만 판단하시오.

JSON 형식으로 응답: {"action_id": "A1", "impact": +1, "rationale": "..."}
```

**Task 3: 시대 민감도 테스트 (축적 편향 잔존 확인)**
```
동일한 정책 행위를 2003년에 시행한 경우와 2023년에 시행한 경우,
적응적 거버넌스 관점에서 평가가 달라지는가? 달라진다면 그 이유는?

JSON 형식으로 응답: {"changes": true/false, "reason": "..."}
```

### 앵커링 비네트 (V-Dem 방식, 교정용)

| 비네트 | 내용 | 정답 |
|--------|------|:----:|
| V1 | "정부 X는 전국적 디지털 인프라를 확대하여 전자정부 순위가 상승했다" | T |
| V2 | "정부 Y는 대규모 감염병 실패 후 질병관리체계를 전면 개편했다" | I |
| V3 | "정부 Z는 시민참여 플랫폼을 운영했으나 접수된 제안의 정책 반영률이 전임 대비 급감했다" | D |

비네트 정확도에 따라 모델별 가중치(1.0/0.7/0.3/0.0) 배정.

### 운용 파라미터
- 5개 모델 × 3라운드 (기존 5라운드에서 축소, Task 세분화로 보상)
- Temperature: 0.3
- 모델 간 일치도: Krippendorff's alpha 보고
- 기존 `data/scripts/ai_scoring_v2.py` API 함수 재활용

### 우선 검증 대상 (7개 항목)
연구자 분류에 이론적 논쟁이 있는 항목만 선별:
1. 이재명 Le1(비전 제시) — I vs T 논쟁
2. 이재명 Le3(협력적 리더십) — 결핍이 I인가 D인가
3. 이재명 D4(AI/디지털) — AI기본법이 I인가 T인가
4. 윤석열 F3(반응성) — 국민청원 폐지가 I인가
5. 문재인 D2(정책 실험) — 규제샌드박스가 I인가 T인가
6. 이명박 Re1(재정 자원) — 4대강 예산이 적응 역량인가
7. 박근혜 F1(합법성) — 국정농단이 D인가 E인가

## 7. AI 재평가 실행 계획

### 기존 방식 (폐기)
```
프롬프트: "노무현 정부의 적응적 거버넌스 역량을 ACW 22기준으로 점수화하라"
→ 축적 편향 재생산
```

### 신규 방식 (3층 구조)
```
Step 1: Action Inventory 작성 (제1층) — 수작업 + 검증 데이터
Step 2: 연구자가 T/I/D/E 코딩 (제2층)
Step 3: I/D 항목을 블라인드 처리하여 5개 AI 모델에 투입 (제3층)
Step 4: AI 검증 결과와 연구자 코딩의 일치도 산출
Step 5: 불일치 항목 재검토
```

### 필요 스크립트
- `data/scripts/blind_coding.py` (신규 작성 필요)
  - `ai_scoring_v2.py`의 API 함수 재활용
  - 블라인드 프로파일 생성기
  - T/I/D/E 분류 검증기
  - 앵커링 비네트 교정기

## 9. 임기 기간 정규화 및 시대 상대 평가 (2026-03-27 추가)

### 9.1 정규화 원칙 (전문가 패널 만장일치)

- **총점을 주된 비교 지표**로 사용 (ACW는 축적적 역량 = stock)
- 연평균은 보조 참고 지표로만 제시
- 자초한 임기 단축(계엄/국정농단)은 **내재적 벌칙**으로 이미 반영됨
- 현직 정부(이재명)는 **잠정 평가**로 표기

### 9.2 시대 상대 평가 3관점

| 관점 | 질문 | 적용 |
|------|------|------|
| 절대 평가 | 적응적 역량이 얼마인가? | ACW 총점 |
| 시대 상대 평가 | 당시 과제에 적절히 대응했는가? | 시대적 과제 대응 수준 |
| 선제적 대응 평가 | 미래 과제를 앞서 준비했는가? | 후속 정부에 대한 제도적 유산 |

### 9.3 행위 기원(Origin) 분류

| 유형 | 정의 | 점수 조정 |
|------|------|----------|
| 자체 입법 | 해당 정부가 발의·통과 | 없음 (해석 투명성용) |
| 전임 이행 | 전임 입법, 해당 정부 시행 | 없음 (이행도 역량) |
| 제도 확대 | 기존 제도 확대·강화 | 없음 |
