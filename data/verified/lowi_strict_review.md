# 적응적(Adaptive) 도구 엄격 재분류 검토

## 목적

기존 `lowi_classification.md`에서 "적응적여부 = O"로 분류된 전체 32건에 대해
Gupta et al.(2010) 및 정책설계(policy design) 문헌의 **엄격한 기준**을 적용하여 재검토한다.

## 엄격 기준 (STRICT Criteria)

| 유형 | 정의 | 핵심 판별 |
|------|------|----------|
| 실험적(Experimental) | 시범사업, 파일럿, 실증, 테스트베드 | 정책을 전면 시행 전 **테스트**하는가? |
| 샌드박스(Sandbox) | 규제 유예 기반 혁신 실험 | 규제를 **유예**하여 실험 공간을 만드는가? |
| 데이터 기반 의사결정 | 데이터·AI를 정책 **의사결정 과정** 자체에 활용 | 거버넌스 설계에 데이터 피드백이 내장되어 있는가? |
| 반복적 수정 | 단계적, 점진적, 재검토, 갱신 | 정책이 피드백에 따라 **수정·갱신**되도록 설계되었는가? |

### 핵심 구분: 적응적 도구가 **아닌** 것

- "디지털 산업 육성" = 산업정책 (배분) ≠ 적응적 거버넌스
- "AI·반도체 투자" = 기술투자 (배분) ≠ 적응적 도구
- "디지털 인재 양성" = 교육정책 (배분) ≠ 적응적 도구
- "사이버안보" = 안보정책 (규제) ≠ 적응적 도구
- "디지털 금융" = 핀테크 산업정책 ≠ 적응적 도구
- "AI 강군" = 군사 현대화 ≠ 적응적 도구

**판별 테스트**: 이 과제는 적응적 **메커니즘**(실험, 샌드박스, 데이터 피드백, 반복적 수정)을 **거버넌스 설계**에 사용하는가? 아니면 단지 "디지털/AI/데이터"라는 **키워드**가 산업·투자 맥락에 포함된 것인가?

## 분석적 함의

### 1. 기존 분류의 과대추정 문제

기존 분류는 과제명에 "AI", "디지털", "데이터", "스마트", "ICT", "사이버" 등의 **키워드가 포함**되면 적응적으로 분류하는 방식이었다. 그 결과 32건이 적응적으로 분류되었으나, 엄격 기준 적용 시 **28건(87.5%)이 탈락**하여 4건(12.5%)만 유지되었다.

### 2. "디지털 산업 육성 ≠ 적응적 거버넌스"

탈락 28건 중 18건(64.3%)이 AI·디지털 **산업 육성**(배분정책)이었다. 이는 기술 키워드의 존재가 자동으로 적응적 거버넌스를 의미하지 않음을 보여준다. AI·반도체에 투자하는 것(산업정책)과 AI를 정책 의사결정에 활용하는 것(적응적 거버넌스)은 근본적으로 다른 차원이다.

### 3. 진정한 적응적 도구의 특성

STRICT-O 4건은 모두 다음 특성을 공유한다:
- **거버넌스 설계 자체**에 적응적 메커니즘이 내장됨
- 정부3.0, 디지털플랫폼정부, AI민주정부 → 정부 **운영 방식**의 데이터 기반 전환
- 규제 재설계 → 규제 **실험** 후 결과에 따른 수정

### 4. 시계열 패턴의 재해석

기존 분류에서는 윤석열(10.0%)·이재명(10.6%) 정부의 적응적 비율이 급증하는 것처럼 보였으나, 이는 대부분 AI·디지털 산업정책의 증가를 반영한 것이었다. STRICT 기준으로는 전 정부에 걸쳐 **0.0~1.6%** 범위로, 한국 국정과제에서 진정한 적응적 거버넌스 도구의 채택은 아직 극히 제한적이다.

### 5. 논문에서의 활용 제안

두 가지 수준의 비율을 모두 보고하는 것이 바람직하다:
- **광의(Broad)**: 디지털/AI 키워드 포함 기준 → 기술 지향성(technology orientation)의 증가를 보여줌
- **협의(Strict)**: 거버넌스 설계 기준 → 진정한 적응적 도구(adaptive governance tool)의 채택 수준을 보여줌

이를 통해 "디지털 키워드의 증가가 반드시 적응적 거버넌스의 확산을 의미하지 않는다"는 중요한 분석적 논점을 제시할 수 있다.
