## 심각도별 이슈 분류

### 🔴 높음 (학술 심사 통과에 직접 위협)

#### H1. 인간 코더 벤치마크 부재
- **발견자**: Lead, R2
- **문제**: 5개 LLM 간 일치도가 인간 전문가와의 일치를 보장하지 않음. 체계적 편향 은폐 위험.
- **권고**: 전체 데이터의 20%에 대해 행정학 전공자 2인의 독립 코딩 수행 → human-AI agreement(Krippendorff α) 보고.
- **최소 대안**: 파일럿 10-20건이라도 수행.

#### H2. 블라인딩 실효성 미검증
- **발견자**: R2
- **문제**: Level 2 블라인딩(정부명/날짜 제거)으로 LLM의 정부 식별을 방지할 수 있는지 실증 없음. "규제자유특구법", "비상계엄"은 강력한 식별 단서.
- **권고**: 블라인드 사례에 대한 LLM 역추론 테스트 수행(정확도가 chance level 16.7%를 초과하는지 확인). 결과를 한계로 명시.

#### H3. I-와 D의 조작적 정의 불충분
- **발견자**: R1
- **문제**: I-(의도적 축소)과 D(작동 실패) 구분이 사례 수준에서 모호. 성과지표 하락이 I- 또는 D에 편입되는 사례 존재.
- **권고**: 판정 트리(decision tree) 제시. I-는 명시적 정책 결정(폐지, 거부권, 예산 삭감)으로 한정. D는 법·조직 유지+집행 실패로 한정. 성과지표 단독 근거 금지.

### 🟡 중간 (보강 시 통과 가능)

#### M1. Hall 가중치(1/2/3) 선형 가산
- **발견자**: R1
- **문제**: Hall의 서열 구분을 등간 척도처럼 사용. 3차수 단일 사건이 총점 과도 좌우.
- **권고**: 1/2/3 외에 1/3/5 또는 사건 수 단순 집계의 민감도 분석 병행.

#### M2. 코딩 단위 혼합(행위 vs 성과지표)
- **발견자**: R1
- **문제**: WGI 잔차, EGDI 순위 같은 결과지표가 I/I-/D에 편입되어 독립변수-종속변수 혼재.
- **권고**: 코딩 대상을 "관찰 가능한 정부 결정"으로 한정. 성과지표는 외부 준거로 분리.

#### M3. 이재명 CRS 비대칭
- **발견자**: R3, Lead
- **문제**: 타 정부 3건 vs 이재명 4건. Case 19(정권 공백기 극복), Case 20(경제 역성장 극복)이 "위기 대응"의 개념 확장. 포함/제외 시 CRS 3.47→3.04로 변동.
- **권고**: 3건/4건 양쪽 결과를 병렬 보고.

#### M4. Lowi STRICT 변별력 부족
- **발견자**: R3, Lead
- **문제**: 595건 중 4건만 STRICT 통과. 3개 정부가 0건. Borda에서 변별 불가.
- **권고**: "보조 지표"로 격하하거나, Borda Count에서 제외하고 별도 분석.

#### M5. T/I/D/E ↔ AGLI 지표 간 중복
- **발견자**: R3
- **문제**: 동일 정책 행위(법률 제정)가 T/I/D/E 가중합과 AGLI Flow에 동시 기여 → 이중 가산 효과.
- **권고**: Spearman 순위 상관 보고. 높은 상관 시 Borda에서 하나 제외 또는 통합.

#### M6. Borda Count vs 비가산 원칙 논리 긴장
- **발견자**: R3
- **문제**: "총점을 산출하지 않는다"면서 Borda로 종합 순위를 제시하는 것은 사실상 가산적 종합.
- **권고**: Borda를 "탐색적 안정성 확인"으로 명시적 격하. "종합 순위"라는 표현 회피.

#### M7. AGLI의 IAD 차용 과장
- **발견자**: R1
- **문제**: "Ostrom IAD 기반"이라는 표현이 실제 차용 범위(rules-in-form/use gap)보다 넓은 인상.
- **권고**: "IAD의 rules-in-form/rules-in-use 구분을 제한적으로 차용한 법제도 운영 지표"로 수정.

#### M8. HCX-007 민감도 분석 부재
- **발견자**: R2 (OpenCode 확인)
- **문제**: HCX-007의 체계적 차이가 보고되나, 포함/제외 시 결과 변동이 분석되지 않음.
- **권고**: 5모델 vs 4모델(HCX-007 제외) 비교를 부록에 제시. leave-one-model-out 진단 추가.

#### M9. 신뢰도 척도 수준 불일치 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: ACW(서열/등간적 척도)에 Fleiss' kappa(명목형 전제)를 사용하여 정보 손실 및 해석 왜곡 위험.
- **권고**: ACW·제도 점수에는 Krippendorff's α(interval/ordinal) 또는 ICC(2,k)를 병행 제시. 명목형 kappa는 보조로 유지.

#### M10. 0점 억제 규칙의 잠재 편향 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: multi_round_scoring.py의 "판단유보 금지" 지침 + 0점 과다 시 검증 실패 규칙이 결합되어 양극화된 점수 분포를 유도할 위험.
- **권고**: 0점 억제의 영향을 확인하기 위해 억제 전/후 분포를 비교 보고.

#### M11. temperature 비일관 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: ai_scoring_v2.py(T=0.3)과 blind_coding.py(T=0.2)의 temperature 상이. CLOVA thinking 모드도 라인 간 불일치(medium vs off).
- **권고**: 전 스크립트 동일 temperature(0.2 권장) + thinking 모드 정렬.

#### M12. 이재명 컨텍스트 비대칭 주입 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: build_acw_prompt에서 이재명 정부에만 지방 경력 컨텍스트(local_context)를 조건부 주입 → 정보 비대칭에 따른 체계적 편의 가능성.
- **권고**: 전 정부에 동등한 컨텍스트 주입 원칙 수립, 또는 컨텍스트 비주입 일원화.

#### M13. 재시도 정책의 선택 편향 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: JSON 파싱 실패 시 재시도하는 구조가 '유효성 통과'에 유리한 샘플을 선택할 가능성.
- **권고**: 1차 응답과 최종 채택 응답을 모두 저장하고, 형식 실패와 내용 실패를 구분 처리.

#### M14. CRS 블라인딩 수준 불충분 (NEW — OpenCode)
- **발견자**: R2 (OpenCode)
- **문제**: crisis_scoring.py는 대통령명만 그리스 문자로 치환(Level 1 수준). blind_coding.py 대비 블라인딩이 현저히 약함.
- **권고**: crisis_scoring.py에도 Level 2 블라인딩 규칙(연도 일반화, 고유사건명 치환) 이식.

#### M15. Lowi STRICT 과제명 기반 분류의 거짓 음성 (NEW — Copilot)
- **발견자**: R3 (Copilot)
- **문제**: 문재인 정부의 규제자유특구법, 국민청원 시스템 등은 명백한 적응적 도구이나, 국정과제명에 해당 단어가 없어 STRICT-X로 탈락. 과제명 기반 분류가 거짓 음성(false negative)을 과다 생성.
- **권고**: 세부 실천과제 또는 법제도 산출물 수준에서 재분류하거나, 중간 기준(Moderate) 도입 검토.

#### M16. 사후 평가 편향(Hindsight Bias) (NEW — Copilot)
- **발견자**: R3 (Copilot)
- **문제**: AI 모델은 사전 학습 데이터에 언론 보도, 학계 평가, 여론이 포함되어 있어, 당시의 제약 ���건이 아닌 사후적 정보를 기준으로 채점했을 가능성.
- **권고**: 한계로 명시적 인정. 가능하면 사전 학습 데이터 컷오프를 통제.

#### M17. ACW 프로파일 범주화 기준 불명확 (NEW — Copilot)
- **발견자**: R3 (Copilot)
- **문제**: "강점/양호/중립/혼재/약점" 범주 판정 기준이 미명시. 양수/음수 어떻게 판정하는지, +1의 기준이 무엇인지 불투명.
- **권고**: "해당 차원의 4개 기준 중 양수(+) 개수가 4개=강점, 3개=양호" 등 정량적 기준 명시.

#### M18. CRS 사례 선정 기준 투명성 부족 (NEW — Copilot)
- **발견자**: R3 (Copilot)
- **문제**: 수십 건의 위기 후보 중 3-4건을 선정한 기준이 불명확. 예: 노무현 정부에서 대구 지하철 참사(192명 사망)가 제외된 이유 불명.
- **권고**: 위기 사례 선정 프로토콜 공개(언론 보도량, 사상자 규모, 공식 재난 선포 여부 등).
