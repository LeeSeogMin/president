## 0. 설계 원칙

블라인드 코딩은 **1차 측정 도구가 아니다**. 이미 완료된 다중 AI 스코어링(5모델x5라운드)의 결과를 사후적으로 검증하는 **삼각검증의 세 번째 레이어**이다.

```
Layer 1: 다중 AI 스코어링 (5모델 합의) ← 1차 측정
Layer 2: 모델간 수렴도 분석 (ICC, 범위) ← 내적 일관성
Layer 3: 블라인드 코딩 (본 프로토콜) ← 귀인 타당성 검증
```

**핵심 질문**: 연구자의 T/I/D/E 분류가 타당한가? 즉, 특정 정부의 높은/낮은 ACW 점수가 대통령의 의도적 행위(I)에 기인하는 것인지, 아니면 시대적 추세(T)나 외부 충격(E)의 산물인지를 독립적으로 검증한다.

## 1. T/I/D/E 귀인 분류 체계

### 1.1 분류 정의

| 코드 | 명칭 | 정의 | 비교 포함 여부 |
|------|------|------|:---:|
| **T** | 추세(Trend) | 어떤 대통령이든 일어났을 시대적 축적. 예: IT 인프라의 자연적 발전, GDP 성장에 따른 재정 확대 | 제외 |
| **I** | 의도(Intentional) | 대통령의 의도적 정책 결정과 집행. 예: 특정 제도 신설, 조직 개편, 예산 재배분 | **포함** |
| **D** | 표류(Drift) | 제도가 존재하나 작동하지 않음. 예: 법은 있으나 이행 안 됨, 위원회가 있으나 형식적 운영 | **포함** |
| **E** | 외부(External) | 정부 통제 밖의 외부 충격. 예: 글로벌 금융위기, 팬데믹, 국제 규범 변화 | 제외 |

### 1.2 비교 포함 논리

- T와 E를 제외하면, 정부별 ACW 점수 차이는 **대통령의 의도적 행위(I)와 제도적 방치(D)**에 의해 설명된다.
- 이 분류가 타당해야 "이재명 정부 ACW 1위"라는 결과가 시대적 추세가 아닌 정책적 선택의 결과라고 주장할 수 있다.

## 2. 블라인드 수준 설계

### 2.1 세 수준의 블라인딩

| 수준 | 제거 대상 | 잔존 단서 | 식별 위험 |
|------|----------|----------|:---:|
| **Level 1** | 이름, 정당, 재임 기간 | 정책 내용, 시대 배경, 사건명 | 높음 |
| **Level 2** | + 구체적 사건명, 날짜, 법률명 | 정책 방향성, 구조적 특성 | 중간 |
| **Level 3** | + 정책 묶음 해체 (기준별 단독 제시) | 단일 행위의 구조적 특성만 | 낮음 |

### 2.2 수준별 실행 가능성 평가

**Level 1 (이름 제거만)**: AI 모델은 "규제샌드박스법 제정"이라는 문구만으로 시기를 특정하고, "문재인 정부"를 높은 확률로 식별한다. 한국 정치사의 사건-정부 연결이 고유하기 때문이다. **검증용으로 부적합**.

**Level 2 (맥락 추상화)**: "정부 A는 기존 규제 체계에 유연성을 도입하는 법률을 제정했다"와 같이 추상화하면 식별 확률이 크게 감소한다. 그러나 정책 조합 패턴(예: "디지털 전환 + 규제 완화 + 시민참여 강화")이 특정 정부를 연상시킬 수 있다. **조건부 적합**.

**Level 3 (기준별 단독)**: ACW 22개 기준 각각에 대해 독립적으로 제시하면, 정책 묶음 효과가 사라져 식별이 어려워진다. 그러나 맥락 부재로 인해 AI가 T/I/D/E를 판단할 근거 자체가 희박해질 위험이 있다. **T/I 판별에는 부적합, D 판별에는 적합**.

### 2.3 권장 전략: Level 2 + Level 3 하이브리드

```
Phase 1: Level 2로 정부 프로파일 전체를 제시 → T/I/D/E 귀인 판정
Phase 2: Level 3로 쟁점 기준만 단독 제시 → D(표류) 여부 집중 검증
Phase 3: 식별 테스트 (아래 §4) → 블라인딩 성공 여부 확인
```

## 3. 귀인 검증 절차

### 3.1 검증 대상 선정

22개 ACW 기준 x 6개 정부 = 132개 판정 중, 블라인드 검증이 필요한 것은 **연구자의 분류에 이론적 논쟁이 있는 항목**이다.

**우선 검증 대상**:

| 기준 | 정부 | 연구자 분류 | 논쟁점 |
|------|------|:---:|--------|
| Le1(비전 제시) | 이재명 | I | 임기 1년차, honeymoon effect 가능성 |
| Le3(협력적 리더십) | 이재명 | D | 의도적 비협력인가, 구조적 한계인가 |
| D4(AI/디지털) | 이재명 | I+T | AI 기본법은 전 정부 입법, 이재명은 집행 단계 |
| F3(반응성) | 윤석열 | D | 제도 존재했으나 작동 안 됨 vs. 의도적 무시(I) |
| D2(정책 실험) | 문재인 | I | 규제샌드박스는 시대적 추세(T)인가 |
| Re1(재정 자원) | 이명박 | T+I | 글로벌 재정 확대 추세 vs. 녹색 뉴딜 의도적 투자 |
| F1(합법성) | 박근혜 | D→I(부정) | 제도적 표류인가, 적극적 침해인가 |

### 3.2 블라인드 프로파일 생성 규칙

**Step 1**: 정부 식별자 치환
```
노무현 → 정부 A    이명박 → 정부 B    박근혜 → 정부 C
문재인 → 정부 D    윤석열 → 정부 E    이재명 → 정부 F
```
**주의**: 순서를 무작위로 섞는다. 매 세션마다 재배정한다.

**Step 2**: 시간 표지 제거
- 절대 연도 → "임기 N년차"
- 특정 사건명 → 추상화 ("대규모 감염병 위기", "대통령 탄핵")
- 법률명 → 기능적 기술 ("AI 관련 기본 법률 제정")

**Step 3**: 정당/이념 표지 제거
- "진보/보수" → 삭제
- "야당 출신", "정권 교체" → 삭제
- 정당명 → 삭제

## 4. 식별 테스트 (Identification Test)

### 4.1 설계

블라인드 코딩 완료 후, **동일 세션** 또는 **별도 세션**에서 다음 프롬프트를 투입한다.

**기준 확률**: 6개 정부 무작위 배정 시 우연 식별 확률 = 1/6 = 16.7%

### 4.2 판정 기준

| 식별률 | 판정 | 조치 |
|:---:|------|------|
| ≤16.7% (1/6) | 블라인딩 완전 성공 | 블라인드 코딩 결과를 그대로 사용 |
| 17-33% (1-2/6) | 부분 성공 | 식별된 정부의 블라인드 결과를 별도 보고, 나머지 사용 |
| 34-50% (2-3/6) | 부분 실패 | Level 3로 재실시, 식별 실패 이유 분석 |
| >50% (3+/6) | 블라인딩 실패 | 블라인드 코딩 포기, 대안 검증(앵커링 비네트만 사용) |

### 4.3 식별 실패 시 대응 전략

블라인딩이 실패하더라도 연구의 치명적 약점이 되지 않는다. 이유:

1. **블라인드 코딩은 Layer 3 검증**이다. Layer 1(다중 AI 합의)과 Layer 2(모델간 수렴도)가 주된 증거이다.
2. 식별 실패 자체가 정보이다: "한국 대통령의 거버넌스 패턴이 충분히 독특하여 익명화가 어렵다"는 것은 **정부별 차이가 실질적으로 존재함**을 반증한다.
3. 대안: 식별 테스트 결과를 투명하게 보고하고, **앵커링 비네트 검증(§5)**으로 보완한다.

## 5. 앵커링 비네트 (Anchoring Vignettes)

### 5.1 V-Dem 접근법 적용 원리

V-Dem 프로젝트의 앵커링 비네트는 코더 간 척도 해석 차이를 교정하기 위해 **정답이 알려진 가상 사례**를 제시한다. 본 연구에서는:

- 실제 국가 사례를 변형하여 **T/I/D/E 정답이 명확한 참조 사례** 3개를 설계한다.
- AI 모델이 비네트를 정확히 분류하면, 실제 한국 정부 사례에 대한 분류도 신뢰할 수 있다.
- 비네트 정확도가 낮으면, 해당 모델의 한국 사례 분류를 할인(discount)한다.

### 5.2 비네트 설계 원칙

1. **정답 명확성**: 해당 분야 전문가 3인이 동의할 수 있는 수준의 명확한 T/I/D/E 분류
2. **현실성**: 한국 행정 맥락과 유사하되, 실제 정부와 혼동되지 않을 것
3. **난이도 분포**: 쉬운(T/E) 1개 + 중간(I) 1개 + 어려운(I vs D 경계) 1개

### 5.3 비네트 1: "디지털 인프라 확대" (정답: T — 추세)

```
[비네트 V1]

가상국 K의 정부 X (임기 5년)

배경: 정부 X 취임 시점에 K국의 인터넷 보급률은 85%였다. 정부 X는 "디지털 강국"
비전을 선포하고 디지털 인프라 확대 예산을 전 정부 대비 15% 증액했다.
임기 말 인터넷 보급률은 92%에 도달했다.

그러나:
- K국의 인터넷 보급률은 정부 X 이전 10년간 연평균 2%p씩 증가해왔다.
- 주변 국가들도 동일 기간 유사한 보급률 증가를 보였다.
- 15% 예산 증액분은 인플레이션 조정 시 실질 5% 증가에 그쳤다.

질문: 정부 X 임기 중 인터넷 보급률 7%p 상승은 정부의 의도적 행위(I)인가,
시대적 추세(T)인가?
```

**정답**: **T (추세)**. 기존 추세선(연 2%p)과 일치하며, 국제적 보편 추세이고, 실질 예산 증가분이 미미하다. 대통령 비전 선포는 "라벨링"에 불과하다.

### 5.4 비네트 2: "위기 대응 체계 구축" (정답: I — 의도적)

```
[비네트 V2]

가상국 K의 정부 Y (임기 5년)

배경: 정부 Y 취임 직후 대규모 산업재해가 발생했다(광산 붕괴, 200명 사상).
정부 Y는 다음 조치를 취했다:

1. 기존 분산된 3개 안전 기관을 통합하여 "국가안전청" 신설 (조직법 개정)
2. 안전 관련 예산을 GDP 대비 0.3%에서 0.8%로 인상 (3년에 걸쳐)
3. 민관 합동 안전점검위원회를 법정 기구로 격상 (기존은 자문기구)
4. 산업안전 기준을 OECD 최고 수준으로 상향 (37개 시행령 개정)

이전 정부들은 유사한 재해 후 임시 대책을 발표했으나, 조직 신설이나
법정 기구 격상까지는 진행하지 않았다.

질문: 정부 Y의 안전 거버넌스 강화는 T/I/D/E 중 어디에 해당하는가?
```

**정답**: **I (의도적)**. 외부 충격(E)이 계기이나, 이전 정부들이 동일 충격에 임시 대책만 내놓은 것과 대비하여, 조직 신설/예산 대폭 증액/법정 기구 격상은 의도적 제도 설계이다. 충격 자체는 E이나, **대응의 수준과 영속성이 I를 결정**한다.

### 5.5 비네트 3: "참여 제도의 형식화" (정답: D — 표류)

```
[비네트 V3]

가상국 K의 정부 Z (임기 5년)

배경: 정부 Z는 전임 정부가 도입한 "국민정책참여 포털"을 승계했다.
이 포털은 법적 근거가 있으며(시민참여촉진법 제12조), 연간 운영 예산도
배정되어 있다.

정부 Z 임기 중:
- 포털은 계속 운영되었다 (서버 가동, 접수 기능 유지)
- 그러나 접수된 제안 중 정책 반영률이 전 정부 23%에서 4%로 하락했다
- 제안 검토위원회는 5년간 3회만 소집되었다 (전 정부: 연 4회)
- 포털 방문자 수는 전 정부 대비 70% 감소했다
- 정부 Z는 포털 폐지를 선언하지 않았고, 별도 참여 채널도 신설하지 않았다

질문: 정부 Z의 시민참여 수준은 T/I/D/E 중 어디에 해당하는가?
```

**정답**: **D (표류)**. 제도가 법적으로 존재하고 예산도 배정되어 있으나, 실질적으로 작동하지 않는다. 의도적 폐지(I-부정)가 아니라 방치에 의한 형식화이다. 이것이 D의 전형적 패턴이다.

### 5.6 비네트 채점 기준

| 모델 | V1 정답 | V2 정답 | V3 정답 | 정확도 | 신뢰 가중치 |
|------|:---:|:---:|:---:|:---:|:---:|
| (모델명) | T? | I? | D? | ?/3 | ? |

**가중치 산정**:
- 3/3 정확: 가중치 1.0 (해당 모델의 블라인드 분류를 전적으로 신뢰)
- 2/3 정확: 가중치 0.7 (오류 항목의 유형에서 해당 모델 할인)
- 1/3 정확: 가중치 0.3 (해당 모델의 블라인드 분류를 참고만)
- 0/3 정확: 가중치 0.0 (해당 모델의 블라인드 분류를 사용하지 않음)

## 7. 샘플 블라인드 프로파일

### 7.1 원본 (이재명 정부, Le3 협력적 리더십)

> **이재명 정부 (2025-)**: Le3(협력적 리더십) 합의 점수는 6개 정부 중 일관적으로 최저 수준이다. 5개 모델 전원이 이 기준에서 상대적으로 낮은 점수를 부여했다. 국정과제 추진 시 야당과의 협의보다 행정명령 중심의 추진 방식을 선호하며, 이해관계자 협의 과정이 형식적이라는 평가를 받았다. 대통령의 강한 추진력은 Le2(기업가적 리더십)에서 높은 점수로 이어졌으나, 그 이면으로 Le3에서는 비협력적 의사결정이라는 평가가 나타났다.

### 7.2 Level 2 블라인드 (AI가 실제로 보는 텍스트)

```
정부 Gamma의 행위 기술 — 기준: Le3(협력적 리더십)

정부 Gamma는 취임 초기부터 강한 정책 추진력을 보였다. 주요 국정과제의
경우, 입법부와의 사전 협의보다 행정부 단독의 추진 방식을 선호하는 경향이
관찰되었다. 이해관계자 협의 절차는 법적으로 존재하나, 실질적 반영률이
낮다는 평가가 있다.

5개 독립 평가 모델이 이 정부의 Le3를 평가한 결과, 전원이 다른 기준
대비 상대적으로 낮은 점수를 부여했다. 반면, 같은 정부의 Le2(기업가적
리더십)는 높은 점수를 받았다.

추가 정보:
- 이 정부는 종합 적응역량 평가에서 상위권에 위치한다
- Le3를 제외한 나머지 5개 ACW 차원은 전반적으로 양호하다
- 이 정부의 임기는 비교적 최근에 시작되었다
```

### 7.3 블라인드 프로파일에서 제거/변환된 요소

| 원본 요소 | 블라인드 처리 |
|----------|-------------|
| "이재명 정부 (2025-)" | "정부 Gamma" |
| "국정과제" | "주요 국정과제" (일반적 표현이므로 유지) |
| "야당과의 협의" | "입법부와의 사전 협의" |
| "행정명령 중심" | "행정부 단독의 추진 방식" |
| "대통령의 강한 추진력" | "강한 정책 추진력" |
| "5개 모델 전원이" | "5개 독립 평가 모델" (유지) |
| (이재명의 검찰 출신 배경 등 암시 제거) | (해당 없음) |

### 7.4 식별 위험 평가

이 블라인드 프로파일에서 AI가 이재명 정부를 식별할 수 있는 잠재적 단서:

1. **"임기가 비교적 최근에 시작"** → 윤석열 또는 이재명으로 범위 축소 (위험: 중)
2. **"종합 적응역량 상위권 + Le3만 낮음"** → 이재명 정부의 독특한 패턴 (위험: 중-높)
3. **"Le2 높고 Le3 낮음"** → 기업가적이나 비협력적 리더십 패턴 (위험: 중)

**대응**: "최근 시작" 문구를 삭제하고, ACW 종합 순위 정보를 제거하면 식별 위험이 크게 감소한다.

## 8. 실행 파이프라인

### 8.1 단계별 실행 순서

```
Step 1: 앵커링 비네트 교정 (§6.2)
  → 5개 모델에 비네트 3개 제시
  → 모델별 정확도 산출, 가중치 배정

Step 2: 블라인드 프로파일 생성
  → 연구자가 §3.2 규칙에 따라 7개 쟁점 항목의 블라인드 프로파일 작성
  → 정부 라벨을 무작위 배정 (파이썬 random.shuffle 사용)

Step 3: T/I/D/E 블라인드 분류 (§6.1)
  → 5개 모델에 각 블라인드 프로파일 제시
  → 모델당 1회 (반복 측정 불필요 — 이미 Layer 1에서 5라운드 완료)

Step 4: 식별 테스트 (§6.3)
  → 동일 세션에서 정부 식별 시도
  → 식별률 산출, §4.2 기준에 따라 판정

Step 5: 블라인드 해제 및 비교 (§6.4)
  → 실제 정부명 공개
  → 블라인드 분류 vs. 연구자 분류 일치율 산출

Step 6: 최종 보고
  → 일치율, 불일치 항목, 식별 테스트 결과를 통합하여
  → 본문 또는 부록에 "블라인드 검증 결과" 절 삽입
```
