# Policy Lag 정량 분석 데이터

> **근거 프레임워크**: OECD (2021). *Government at a Glance 2021: Government Agility*. OECD Publishing.
> 정책 지연(Policy Lag)의 3단계 분류: 인지 지연(Recognition Lag), 결정 지연(Decision Lag), 실행 지연(Implementation Lag)

## 1. 측정 방법론

### 지연 유형 정의

| 유형 | 정의 | 측정 기준점 |
|------|------|------------|
| **인지 지연** | 문제 발생 → 정부의 공식적 문제 인지 | 최초 사건 → 정부 첫 공식 발표/회의 |
| **결정 지연** | 문제 인지 → 정책 결정(법률/시행령/대통령령 등) | 공식 인지 → 최종 정책 결정문 확정 |
| **실행 지연** | 정책 결정 → 실제 집행 시작 | 결정 확정 → 국민에 실질적 영향 시작 |
| **총 지연** | 문제 발생 → 실제 집행 시작 | 최초 사건 → 실질적 집행 |

### 데이터 소스
- 관보(대한민국 관보 원문)
- 정부 보도자료(정책브리핑)
- 국회 회의록(의안정보시스템)
- 질병관리청 상황 보고
- 기획재정부 재정 동향

## 4. 데이터 한계점

1. **비교 가능성 제약**: 각 위기의 성격·규모·복잡성이 상이하여 단순 일수 비교에 한계
2. **이재명 정부 데이터**: 대통령 임기 초반으로 비교 가능한 위기 사례 제한적, 지자체장 시기 데이터로 보완
3. **총 지연의 해석**: 인지 지연 0일이 반드시 우수한 것은 아님(위기 성격에 따라 인지 시점 자체가 상이)
4. **결정 품질 vs 속도**: 빠른 결정이 반드시 좋은 결정은 아님. 속도와 품질의 trade-off 미반영
5. **날짜 기준의 자의성**: "공식 인지"의 기준이 정부마다 상이할 수 있음
